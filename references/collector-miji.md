# 抖音作者主页采集秘籍

这份秘籍用于 Codex 中通用执行抖音作者主页采集：先明确目标集合，再采集作者信息、作品、评论、媒体，最后写入飞书并生成可审计报告。它不是一次事故复盘，而是之后所有人都要遵守的操作规范。

## 一、执行铁律

1. **先定义目标集合，再采集。** 用户说“第 4-8 条”时，必须确认它属于哪个集合：主页原始顺序、去掉置顶后、还是置顶集合。
2. **不要用别的集合补数。** 例如“置顶集合第 4-8 条”如果当前只看到 3 条置顶，目标结果就是 0 条；必须写短缺报告，不能拿非置顶作品补。
3. **作者信息是必采项。** 作者表必须包含关注数、粉丝数、获赞数、作品数、抖音号、简介和主页链接。采不到就留空并写失败原因，不允许用 0 伪装。
4. **有头 Chrome 优先。** 使用用户已登录的可见 Chrome。不要默认切 Chrome Dev，不要开多个 CDP 窗口，不要关闭用户已有窗口。
5. **不碰登录凭证。** 禁止读取、打印、导出、保存 cookie、localStorage、sessionStorage、token、密码、Chrome profile。
6. **DOM 只是降级路径。** 如果浏览器控制面只支持只读 DOM，必须标记“降级采集”；最终完整采集应优先走登录态浏览器上下文接口。
7. **飞书必须可审计。** 除数据表外，必须写 `目标筛选报告`，记录目标规则、集合数量、目标数量和短缺/失败原因。

## 二、目标集合定义

| 用户说法 | 集合定义 | 切片 |
|---|---|---|
| 主页原始顺序第 4-8 条 | 主页/API 返回的全部作品原始顺序 | `works[3:8]` |
| 去掉置顶后第 4-8 条 | 过滤掉 `is_top=true` 或可见 `置顶` 标签后的作品 | `non_pinned[3:8]` |
| 置顶集合第 4-8 条 | 仅包含 `is_top=true` 或可见 `置顶` 标签的作品 | `pinned[3:8]` |

判断置顶时优先用接口字段 `is_top`；没有接口字段时，用作品卡片内可见 `置顶` 标签辅助判断。不要把“排在顶部”直接等同于“置顶”。

## 三、标准流程

1. 创建 `runs/<run_id>/`。
2. 写 `run_config.json`：作者 URL、目标规则、飞书目标、是否采评论、是否采媒体、浏览器模式。
3. 连接已登录有头 Chrome，确认不是登录页、风控页或验证码页。
4. 每个作者分别采：
   - profile：昵称、抖音号、简介、关注、粉丝、获赞、作品数；
   - works：`homepage_all`、`pinned`、`non_pinned` 三个有序集合；
   - targets：按用户目标集合和切片选出目标作品；
   - shortage：集合不足时生成 0 个目标作品和一条报告。
5. 对目标作品采直接链接、正文、点赞、评论、收藏、分享、发布时间、评论、回复、媒体。
6. 写飞书：
   - `作者信息`
   - `视频作品`
   - `图文作品`
   - `一级评论`
   - `二级评论`
   - `目标筛选报告`
7. 回读飞书确认数量和关键字段。缺 scope 时状态只能是 `未完成-缺少回读授权`。
8. 写 `final_report.json`，包含 Base 链接、表 ID、计数、失败原因和降级标记。

## 四、飞书表要求

### 作者信息

字段：`用户ID`、`昵称`、`简介`、`关注数`、`粉丝数`、`获赞数`、`作品数`、`主页链接`、`爬取时间`。

### 作品表

视频只写 `视频作品`，图文只写 `图文作品`。链接必须是作品直链：`/video/{id}` 或 `/note/{id}`。建议增加：

- `目标集合`
- `目标集合序号`
- `目标状态`
- `来源`

### 目标筛选报告

每个作者至少一行：

- 作者
- 目标规则
- 目标集合
- 集合总数
- 目标起始序号
- 目标结束序号
- 目标可写数量
- 结论

## 五、常见错误与修正

| 错误 | 后果 | 修正 |
|---|---|---|
| 把“置顶集合第 4-8 条”理解成“去掉置顶后第 4-8 条” | 作品全错 | 重建目标集合，重新采集，旧 Base 标记作废 |
| 用主页整体第 4-8 条替代置顶集合第 4-8 条 | 混入非目标作品 | 只按 `pinned[3:8]` |
| 置顶不足 8 条还硬凑 5 条 | 伪造交付 | 目标为 0，写短缺报告 |
| 作者粉丝数写 0 | 误导用户 | 重新从 profile 段/API 解析，采不到留空并报错 |
| 只写本地 JSON 不写飞书 | 不是交付 | 创建/更新 Base，并给出链接 |
| 只看 batch_create 不回读 | 验证不完整 | 补回读；缺授权则明确缺 scope |

## 六、验收口径

只有满足以下条件才可以说完成：

- 目标规则与用户原话一致；
- 目标作品全部来自指定集合和指定切片；
- 集合不足时没有替代数据；
- 作者 stats 已写入或明确失败；
- 评论/媒体要求有终态；
- 飞书链接、表 ID、本地 run 目录、`final_report.json` 都可追溯；
- 回读通过，或明确状态为 `未完成-缺少回读授权`。

## 七、推荐脚本

使用 `scripts/select_author_targets.py` 做目标集合选择，避免手写切片：

```bash
python scripts/select_author_targets.py \
  --input runs/<run_id>/author_homepages \
  --collection pinned \
  --start 4 \
  --end 8 \
  --output runs/<run_id>/targets.json
```

生成可分发 skill 包：

```bash
python scripts/build_skill_zip.py --output dist/douyin-author-homepage-collector.zip
```
